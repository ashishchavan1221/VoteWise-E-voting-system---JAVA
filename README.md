## 🎯 Problem Statement
Traditional voting systems in colleges and mid-sized organizations often suffer from high operational costs, manual counting errors, and lack of transparency. For students living away from their home constituencies (hostels/PGs), participating in democracy becomes geographically challenging, leading to low voter turnout and a lack of representative leadership.

## 💡 The Solution
VoteWise provides a **military-grade secure** digital alternative:
- **Remote Accessibility:** Cast votes from anywhere (Hostel, Campus, or Home) with 100% digital verification.
- **E2E Encryption:** Votes are cryptographically hashed and secured in the cloud, unalterable once cast.
- **Real-Time Analytics:** Eliminates counting delays with a 2-second live synchronization engine.
- **High-End UI/UX:** A glassmorphic "Obsidian" interface designed for maximum engagement and zero distraction.

---

## 🔥 Key Features

- **🚀 Obsidian Iris Theme:** A premium dark-mode interface with neon Indigo & Fuchsia accents for a high-end SaaS feel.
- **📈 2s Real-Time Polling:** The analytical system polls the Java backend every 2 seconds to ensure all devices are synced.
- **👑 Dynamic Tie-Handling:** Intelligent algorithm detects "Neck & Neck" ties in the winner card with unique visual indicators.
- **📱 100% Cross-Platform Responsive:** Flawless layout on Android, iOS, Windows, macOS, and Linux.
- **🔐 Secure CAPTCHA:** Responsive security code verification to eliminate bot participation.
- **🎨 Vector-Based Logo:** Custom-engineered CSS/SVG branding that scales perfectly without pixelation.

## 🛠️ Technology Stack
- **Frontend:** React.js, Vite, Tailwind CSS, Chart.js 4 (Advanced Matrix Visualization).
- **Backend:** Core Java (Integrated com.sun.net HTTP Server), Maven.
- **Database:** MongoDB Atlas (Cloud Cluster).
- **Communication:** RESTful API with JSON payload exchange.
- **Deployment:** Dockerized Multi-stage environment.

## 🚦 How to Run the Project

### 1. Prerequisites
- **Node.js** (v18+)
- **JDK 17+**
- **MongoDB Atlas Connection**

### 2. Backend Setup
Navigate to the `backend` directory and run the following:
```powershell
# In Windows PowerShell
.\start-server.ps1
```
*The server will start on `http://localhost:8080`. Ensure your credentials are set in environment variables.*

### 3. Frontend Setup
Navigate to the `frontend-react` directory:
```bash
npm install
npm run dev
```
*The app will be live at `http://localhost:5173`.*

### 4. Simplified One-Click Run
You can use the root `run.bat` file to launch both systems simultaneously in a Windows environment:
```cmd
run.bat
```
